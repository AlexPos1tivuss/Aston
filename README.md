# АСТОН — Блок обратной связи

Полнофункциональный модуль обратной связи для сайта IT-компании АСТОН (astondevs.ru). Включает пользовательскую лендинг-страницу с формами обращений и административную панель для управления заявками.

## Описание проекта

Проект реализует два основных пользовательских сценария для неавторизованных посетителей сайта:

- **Сообщение об ошибке** (US C1-17) — форма обратной связи для информирования компании о проблемах
- **Заказ обратного звонка** (US C1-9) — заявка на обратный звонок от специалиста

Все данные сохраняются в PostgreSQL и доступны администратору через панель управления.

## Бизнес-процессы

### BP-1: Сообщение об ошибке (Форма обратной связи)

Форма предназначена для информирования компании о проблемах и не подразумевает ответа пользователю. Данные носят статистический характер.

```

├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  Пользователь               Frontend               Backend         │
│       │                         │                      │            │
│       │  Нажимает "Сообщить     │                      │            │
│       │  о проблеме" в футере   │                      │            │

│       │                         │                      │            │
│       │  Заполняет форму:       │                      │            │
│       │  - ФИО (необяз.)        │                      │            │
│       │  - Категория            │                      │            │
│       │  - Сообщение (20-400)   │                      │            │
│       ├────────────────────────►│                      │            │
│       │                         │                      │            │
│       │                         │ POST /api/v1/        │            │
│       │                         │    feedbacks         │            │
│       │                         ├─────────────────────►│            │
│       │                         │                      │            │
│       │                         │     ┌────────────┐   │            │

│       │                         │     │ • Rate     │   │            │

│       │                         │     │   текста   │   │            │
│       │                         │     └─────┬──────┘   │            │
│       │                         │           │          │            │
```

## База данных

### Таблица `feedbacks`

Хранит сообщения об ошибках от пользователей.

| Столбец      | Тип             | Обязательность | Описание                              |
|--------------|-----------------|----------------|---------------------------------------|
| `id`         | SERIAL (PK)     | Да             | Уникальный идентификатор              |
| `name`       | VARCHAR(50)     | Нет            | ФИО пользователя                      |
| `message`    | TEXT            | Да             | Текст сообщения (20-400 символов)     |
| `category`   | VARCHAR(64)     | Да             | Категория (по умолчанию "Без категории") |
| `timestamp`  | TIMESTAMPTZ     | Да             | Дата и время отправки (ISO 8601)      |
| `created_at` | TIMESTAMPTZ     | Да             | Дата создания записи (автозаполнение) |

### Таблица `callbacks`

Хранит заявки на обратный звонок.

| Столбец        | Тип             | Обязательность | Описание                              |

| `id`           | SERIAL (PK)     | Да             | Уникальный идентификатор              |

## Безопасность и авторизация

### Публичные эндпоинты

- `POST /api/v1/feedbacks` — без авторизации
- `POST /api/v1/callbacks` — без авторизации
- Защита: rate limiting (1 запрос в 3 минуты на IP)

### Административные эндпоинты

- Все `GET` и `PATCH` эндпоинты защищены middleware `adminAuth`
- Проверяется заголовок `x-admin-token` против переменной окружения `ADMIN_SECRET`
- При отсутствии/несовпадении токена: `403 Forbidden`
- При отсутствии `ADMIN_SECRET` на сервере: `503 Service Unavailable`

**Схема передачи токена:**
- Серверная часть: middleware читает `ADMIN_SECRET` из окружения и сравнивает с заголовком `x-admin-token`
- Клиентская часть (админ-панель): переменная `VITE_ADMIN_TOKEN` встраивается в клиентский бандл через Vite; fetch-интерцептор в `main.tsx` автоматически добавляет заголовок `x-admin-token` ко всем запросам к `/api/`
- **Важно:** данная схема предназначена для внутреннего использования. Для продакшен-среды с публичным доступом рекомендуется перейти на серверную аутентификацию (сессии, JWT) без экспонирования секрета в клиентском коде

### Rate Limiting

```
┌─────────────────────────────────────────────────────┐
│              ДВУХУРОВНЕВЫЙ RATE LIMITING             │
├─────────────────────────────────────────────────────┤
│                                                     │
│  Уровень 1: Клиент (localStorage)                  │
│  ┌───────────────────────────────────────────┐      │
│  │ При отправке формы сохраняем timestamp    │      │
│  │ в localStorage. Если прошло < 3 мин —     │      │
│  │ блокируем отправку на клиенте.            │      │
│  └───────────────────────────────────────────┘      │
│                                                     │
│  Уровень 2: Сервер (in-memory Map)                 │
│  ┌───────────────────────────────────────────┐      │
│  │ Map<IP, timestamp>                        │      │
│  │ Если запрос с того же IP < 3 мин назад — │      │
│  │ ответ 429 Too Many Requests.              │      │
│  └───────────────────────────────────────────┘      │
│                                                     │
│  Период: 1 запрос в 3 минуты (180 000 мс)          │
└─────────────────────────────────────────────────────┘
```

## Валидация данных

### Форма обратной связи (Feedback)

| Поле        | Правила                                                   |
|-------------|-----------------------------------------------------------|
| ФИО         | Необязательно, max 50, клиентская фильтрация: только кириллица + пробел + дефис |
| Категория   | Обязательно, из списка: Без категории, Отделение банка, Банкоматы, Сайт |
| Сообщение   | Обязательно, min 20, max 400 символов                     |
| Timestamp   | Обязательно, ISO 8601 UTC                                 |

**Сообщения об ошибках:**

- `< 20 символов` → «Слишком короткий текст (минимальная длина - 20 символов)»
- `> 400 символов` → «Слишком длинный текст (максимальное количество символов - 400)»
- Rate limit → «Превышена частота запросов. Повторите попытку позже.»
- Сервер → «Сервис временно недоступен. Повторите попытку позже»

### Форма обратного звонка (Callback)

| Поле        | Правила                                                             |
|-------------|---------------------------------------------------------------------|
| Имя         | Обязательно, 2-25 симв., первые 2 — кириллица, допустимо: А-Яа-яЁё, пробел, точка, дефис |
| Телефон     | Обязательно, формат +7XXXXXXXXXX, ровно 10 цифр после +7           |
| Дата        | Обязательно, рабочий день (пн-пт), от завтра, не далее 1 месяца вперёд |
| Время       | Обязательно, из списка: 09:00-10:00 ... 17:00-18:00               |

**Сообщения об ошибках:**

- Имя `< 2 букв` → «Имя должно состоять минимум из 2 букв»
- Поле пустое → «Заполните это поле»
- Дата не выбрана → «Выберите корректную дату»
- Rate limit → «Превышена частота запросов. Повторите попытку позже.»
- Сервер → «Ошибка 500: Сервис временно недоступен»
